## 1. 背景与问题

### 1.1 业务背景

医药电商结算/运营场景中，需要将 **主品** 与 **搭配品（副品）** 组合呈现给用户（类似 App「用药组合」卡片），以提升关联购买与体验。

### 1.2 要解决的问题

- **选品**：从大量 SKU 中选出与主品 **医学/场景合理** 的搭配品，避免弱关联凑单。
- **话术**：输出面向 C 端的 **套餐名称 + 一句话卖点**，风格接近真实「用药组合」页，而非后台公文。
- **可控**：支持 **规则引擎兜底**、**百炼（通义）可选**；预算、超时、开关可配置。
- **运营闭环**：库存清单维护、待组货批次上传、组货结果确认与同步、指标与导出。

### 1.3 明确非目标（当前 MVP）

- **定价策略**：组货阶段 **不按毛利优化定价**；换购展示价以 **商品原价（`original_price`）** 为准，促销价由业务系统后续承接。
- **高并发生产级**：当前以 **单机 SQLite + 单进程** 为主；大规模并发需后续迁库与水平扩展。

## 2. 产品目标

| 目标 | 说明 |
|------|------|
| G1 | 运营可独立完成：**库存导入 → 待组货导入 → 一键/规则生成组货 → 查看/导出 → 确认同步**。 |
| G2 | **真可调百炼**（配置 Key/BaseURL/超时），失败或关闭时 **规则引擎可独立出结果**。 |
| G3 | **禁忌与误匹配可拦截**（如儿童牙膏+润滑液、牙膏「益生菌」误判为肠道等），持续可配置规则扩展。 |
| G4 | 提供 **结算侧可调用的推荐 API**，超时降级，不阻断主交易链路（设计目标）。 |

## 4. 功能需求（MVP）

### 4.1 运营后台（`/admin`）

- **库存清单**
  - Excel 上传更新 `products`（按 `sku_id` **覆盖/合并** 同一 SKU）；表头含科室、**主码**（支持「**主码**」「**标品主码**」「产品编码」等，入库 `product_code`）、商品编码、一级类目、产品名称、生产厂家等；**药网 BI 导出表勿用只读解析**（已用常规模式避免少列误报）。
  - 列表 **分页（如每页 50）**、搜索（含主码/商品编码）、**序号列** 便于阅读。
- **智能组货**
  - 上传 **待组货 Excel**（可无独立「价格」列，系统占位或按 GMV/销量推导）。
  - **生成组货方案**：支持 **规则优先**（默认）与 **可选百炼逐条推理**（较慢）。
  - **批次记忆**（同浏览器 sessionStorage 恢复 `batch_id`，刷新后可继续操作）。
  - 结果表字段：**ID、套餐名称、商品 A/B、组货卖点、原因、来源（规则/百炼）、副品原价、参考毛利、状态、确认**。
  - **复制卖点** 按钮（一键复制 C 端一句话）。
  - **导出当前批次 CSV**：列含 ID、套餐名称、商品 A、商品 A 商品编码、商品 B、商品 B 商品编码、组货卖点、商品价格（副品原价）、参考毛利；UTF-8 BOM。
- **预算与消耗**
  - 模型、API Key、Base URL、超时、月预算与 Token 单价、**启用 AI**；保存后即时生效。

### 4.2 组货逻辑（核心）

- **候选池**：默认从 **库存清单** 取 `active=1`；可按主品类目关键词 **预筛 + 上限**，避免全库扫描卡死。
- **规则引擎**
  - 关联度评分、**医学安全表**、**弱关联过滤**、**防单品灌场**（单批次副品占比上限）。
  - **禁忌搭配**：如口腔/家清/儿童主品 **不得** 搭配润滑、避孕套、情趣品牌等（关键词表可扩展）。
  - **话术**：主品功效句 + 搭配品功效句；**牙膏等口腔优先于「益生菌→肠道」** 误判。
  - **套餐名称**：短标题（如「流行性感冒」「补钙维C」），规则 + 百炼均可产出。
- **百炼**
  - OpenAI 兼容接口；JSON 输出含 `selected_sku_id`、`package_name`、`sales_copy` 等；候选 SKU 白名单校验防 hallucination。

### 4.3 对外 API（摘要）

- `POST /api/recommend`：主品 + 候选池 → 推荐结果（供结算页异步调用）。
- `GET /health`：健康检查。
- `GET /docs`：OpenAPI。
- 运营类 `/api/admin/*`、`/api/ops/*`：**生产环境须加鉴权/白名单**（当前 MVP 默认无登录）。

### 4.4 数据与持久化

- **SQLite**（可配置 `APP_DB_PATH`）：`products`、`uploaded_products`、`bundle_recommendations`（含 `package_name`、`main_item_code`、`selected_item_code`、`sales_copy` 等）、`llm_settings`、`ai_usage_logs` 等。
- **商品编码**（导出/展示用）：优先 `products.item_code`，其次主码 `product_code`，否则 `sku_id`。

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 可用性 | 单进程 + 单 DB 文件，避免多 uvicorn 共写导致 `database is locked`。 |
| 性能 | 大批量生成时支持 Top N 限制、候选池上限、前端较长超时；AI 模式明显慢于纯规则。 |
| 安全 | API Key **仅存服务端**；公网部署必须 **HTTPS + 后台保护**（Basic Auth / IP 白名单 / 登录）。 |
| 可观测 | 管理端 KPI、AI 调用与 Token 日志表；服务端日志。 |
| 部署 | 见代码库 `DEPLOY.md`、`DEPLOY_ALIYUN.md`（含阿里云 ECS 步骤）。 |

## 6. 关键交互与验收标准（抽检）

1. 上传库存 + 待组货 → **不勾选 AI** → 生成 → 列表 **有套餐名、卖点无历史「组合针对…家庭常备」套话**（新逻辑 + 读时剥离旧后缀）。
2. **导出 CSV** 字段齐全，Excel 打开无乱码。
3. **儿童牙膏 + 润滑液** 类组合 **不应出现**（禁忌拦截）。
4. 预算页配置 Key 并勾选启用 → **来源可为百炼**（网络与 Key 正常时）。

## 8. 术语表

| 术语 | 含义 |
|------|------|
| 主品 | 待组货批次中的锚定 SKU。 |
| 副品/搭配品 | 从库存候选池中选出的 1 个 SKU。 |
| 套餐名称 | C 端卡片标题级短文案（2～10 字为宜）。 |
| 组货卖点 | C 端一句话说明主品+搭配价值（无策略套话后缀）。 |
| 参考毛利 | 基于副品原价与成本的粗略毛利示意，非财务结算依据。 |
