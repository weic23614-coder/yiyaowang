# 备份约定：把聊天记录写进仓库（给未来的助手与自己）

## 目的

大模型对话**不会自动**进 GitHub；新会话也可能**读不到**旧会话全文。约定在 **`组件化/`** 目录内**每次重要讨论后**落一条可追溯记录，便于续聊与交接。

## 怎么做（推荐）

1. **每次**与 Cursor / 助手结束一段「组件化 / OpenClaw / 小程序 Demo」相关讨论后，在本目录新增：  
   `会话记录-YYYY-MM-DD.md`  
   （若同一天多轮，可加后缀：`会话记录-2026-03-21-2.md`）

2. 文件建议结构：
   - **参与者**、**日期**
   - **用户原话**（可复制粘贴关键消息）
   - **助手结论 / 待办 / 风险**（条目化）
   - **已变更的文件路径**（若有）

3. **摘要文档**（如 `05-...对话备份.md`）可在里程碑时更新；**会话记录**可更口语、更长，不必与 05 重复打磨。

4. 推送：在仓库根目录 `git add 组件化/ && git commit && git push`（勿在同一行命令末尾写 `# 注释`，避免 zsh/git 误解析）。

## 禁止事项

- 勿把 **Token、密码、客户未脱敏数据** 贴进公开仓库；敏感内容用「已省略」占位。
