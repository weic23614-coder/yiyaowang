# 对话记录（Cursor / 助手）

本目录用于**持久化聊天上下文**，避免新会话无法继承旧讨论。与方案正文（如 `B2C代运营/`）分离，专门放**会话原文或忠实整理**。

## 约定（请助手与用户共同遵守）

1. **每次**结束一段重要讨论后，新增一条文件：  
   `YYYY-MM-DD-主题简述.md`  
   同一天多轮可加后缀：`2026-03-21-2-主题.md`。

2. **内容建议包含**  
   - 日期、主题、涉及目录（如 `B2C代运营/`、`其他项目/`）  
   - **用户原话**（可复制粘贴关键条消息）  
   - **助手回复要点**（可全文或分段摘要；若回复极长，用「结构化摘要 + 原文要点列表」避免丢失决策）  
   - **已落地的仓库变更**（文件路径、commit 主题）  
   - **待办 / 待你拍板**（若有）

3. **安全**  
   勿写入 Token、密码、**未脱敏客户名与商业机密**；敏感处用「已省略」。

4. **用户口令「备份」**  
   用户说 **备份**（聊天记录/会话）时，即表示：在本仓 **`对话记录/`** 下新增或更新存档文件，并 **推送到 GitHub** [`weic23614-coder/yiyaowang`](https://github.com/weic23614-coder/yiyaowang) 的 [`对话记录`](https://github.com/weic23614-coder/yiyaowang/tree/main/%E5%AF%B9%E8%AF%9D%E8%AE%B0%E5%BD%95) 目录（`git push origin main`）。

5. **推送**  
   在仓库根目录：`git add 对话记录/ && git commit -m "docs: 对话记录 ..." && git push origin main`
