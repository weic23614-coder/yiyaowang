## 助手结论与要点

### 1. `ERR_CONNECTION_REFUSED`

- 含义：该 IP + 端口**没有进程在监听**（或拒绝连接）。  
- 常见原因：本地服务未启动；或**端口写错**（例如文档写 8088，实际一直用 8089）。

### 2. 服务已起但仍「404」

- 日志若出现 **`GET /...`**：地址栏里路径字面量是 `/...`（省略号被当成路径），**不是**有效路由。  
- 应使用例如：`http://127.0.0.1:8088/admin` 或 **`http://127.0.0.1:8089/admin`**（与启动命令里的 `--port` 一致）。

### 3. 电脑重启后打不通

- **开发用 uvicorn 不会随重启自动启动**，需重新执行启动命令，或使用下文「登录自启动」。

### 4. 端口 8088 vs 8089

- 仓库 `README` / `DEPLOY` 默认示例多为 **8088**；用户习惯 **8089** 时，只要 **`uvicorn --port` 与浏览器 URL 端口一致**即可。

### 5. 本机已落地的脚本与文档（zhinengzuhuo-backup）

| 路径 | 作用 |
|------|------|
| `scripts/start-dev.sh` | 手动开发启动，默认 **8089**，带 `--reload` |
| `scripts/install-login-startup.sh` | macOS 用户 **LaunchAgent**，登录后自动 `uvicorn` 监听 **127.0.0.1:8089**（无 reload，`KeepAlive`） |
| `scripts/uninstall-login-startup.sh` | 卸载上述自启动 |
| `.gitignore` | 增加 `.logs/` |
| `README.md` | §1.3 说明 8089 与自启动 |

**说明**：在 Cursor 沙箱内无法写入 `~/Library/LaunchAgents/`，用户需在**本机终端**执行：

```bash
cd ~/zhinengzuhuo-backup
./scripts/install-login-startup.sh
```

装好后固定打开：**http://127.0.0.1:8089/admin**  
日志目录：`zhinengzuhuo-backup/.logs/`
