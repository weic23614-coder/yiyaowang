# 对话脉络与要点（结构化，非逐字全文）

> 说明：完整逐字聊天记录需用户从 Cursor 导出或复制粘贴补充；此处为助手根据会话整理的**可追溯要点**，便于次日接续。

## 阶段 A：MVP 范围与产品规则

- 用户明确：**智能组货 MVP**，后台要能跑通；**真调用百炼（qwen）**，非纯规则壳。
- **运营后台**：库存表头顺序固定为「科室、产品编码、商品编码、一级类目、产品名称、生产厂家」；列表与组货结果 **每页 50 条**。
- **预算页**：可配置 **API Key、Base URL、超时**，保存后即时生效。
- **业务边界**：只管组货、不管定价策略；换购副品展示用 **`original_price`**；选品侧重关联度与医学安全，弱化按毛利凑数；**防单品灌场**（单批次同一副品占比上限）；弱关联过滤。
- **数据**：待组货 Excel **可无独立价格列**，用 GMV/销量推导或占位，**不能因缺价拒绝上传**。

## 阶段 C：故障排查

1. **`database is locked`**  
   - 处理：超时、WAL、重试；根因强调 **勿多开 uvicorn 共写同一 db**。

2. **上传/生成卡住、页面一直转**  
   - 处理：流式读行、候选池预筛与上限、前端超时与错误提示。

3. **`缺少关键字段：价格`**  
   - 处理：去掉对价格列的硬性要求，支持推导/占位。

4. **端口混乱（8088～8098）**  
   - 建议：**单进程 + 固定端口 + 单一 DB**；日志中曾出现某端口 **生成 30s 超时**，需真实数据验证耗时。

## 阶段 D：未完全闭环

- 用户体感：**点了 AI 组货没反应、费用不变**。  
- 可能原因：请求未完成、多实例锁库、未命中计费/日志路径。  
- 次日应在 **明确端口 + app_runtime.db** 上复现，并查 **`ai_usage_logs`** 与管理端展示。
